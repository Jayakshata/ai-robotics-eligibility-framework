## 1. Definition & Scope
The **software/systems substrate every robot runs on** — the middleware that moves messages between perception, planning, and control; the real-time layer that makes those loops meet deadlines; and the embedded/edge stack that executes it all on-robot under compute, power, and latency budgets. It sits *under* the whole capability stack rather than beside it, and — uniquely in Axis A — its home turf is **Computer Engineering / ECE embedded + computer-architecture + OS + real-time coursework**, not a generic "Robotics" degree; a strong self-taught embedded-Linux/RTOS engineer is a first-class feeder here. **In scope:** ROS/ROS 2 architecture (nodes, topics/services/actions, tf2, lifecycle, executors & callback groups, the RMW abstraction); DDS pub/sub & QoS (Cyclone / Fast DDS / RTI Connext) and the **Zenoh** alternative; real-time systems (RMS/EDF scheduling, WCET, priority inversion/inheritance, schedulability analysis, PREEMPT_RT, RTOS: FreeRTOS/Zephyr/QNX); embedded programming (ARM Cortex-M, bare-metal, drivers, interrupts/DMA, **micro-ROS**); real-time networking & fieldbuses (EtherCAT, CAN/CAN-FD, TSN); zero-copy / GPU-accelerated transport (iceoryx, NITROS); and **edge / on-device AI inference & optimization** (quantization/pruning, TensorRT/TVM/TF-Lite Micro, Jetson) up to **on-robot foundation-model (VLA) inference**. **Out of scope (adjacent A10 siblings & consumers):** formal safety/verification & runtime assurance (real-time is the *enabler*, provable safety is owned there), sensor/actuator calibration & bring-up (calibration/integration niche), training-side MLOps/data pipelines (robotics-MLOps niche), and the perception/control/VLA *algorithms* that run on top (A1/A4/A7 — consumed here, owned there).

## 3. Eligibility Criteria — Acceptable & Highly-Valued Credentials

**Acceptable undergraduate degrees** — Tier 1 (auto): CS (with AI/ML), Robotics Engineering, EE/ECE, AI. Tier 2 (with evidence): Mechanical Engineering, Mechatronics, Maths, Physics. **A10-specific reweighting `✓` (toward systems/hardware and *away* from a generic Robotics degree — the reverse of most Axis-A niches):** **Computer Engineering is the single strongest match** (embedded, computer architecture, OS, digital design are its core — here it is the *headline* degree, not merely CS-equivalent); **EE/ECE rises** (microcontrollers, signals, hardware, real-time are native); **CS-with-a-systems-concentration rises** (OS, computer architecture, distributed systems, networking, compilers) — but a **pure ML/AI-track CS degree does *not* auto-qualify**. **Robotics Engineering / Mechatronics HOLD** — qualify only with the systems spine (RTOS/architecture/systems-programming) on the transcript, since many robotics majors are perception/ML-heavy. **Mechanical Engineering FALLS** (dynamics/actuation are peripheral to middleware/RTOS — the inverse of its Tier-1 rise in Manipulation); **Applied Maths / Physics fall** too — this niche's math is discrete/systems, not continuous/estimation. **Unique feeder `✓`: self-taught / non-degree systems engineers** with deep embedded-Linux, RTOS, C, and concurrency experience are first-class here in a way they are not in theory-heavy niches. *A10 transcript checklist (replaces the general controls-elective line): computer architecture **+** operating systems **+** a real-time/embedded course · C/C++ & concurrency · a scheduling/RTOS **or** ROS 2 middleware project · (for edge-AI roles) a model-optimization / on-device-inference project.*

## 4. Skill Profile (JD-style)

**Required / must-have**
- Strong **C and C++ systems programming** with **concurrency & multithreading**; Python for tooling/glue
- **ROS 2** depth: nodes, topics/services/actions, tf2, lifecycle nodes, **executors & callback groups**, launch, colcon
- **DDS & the RMW layer**: pub/sub, **QoS policies** (reliability/durability/deadline/history), discovery; Cyclone / Fast DDS / Connext; the Zenoh alternative
- **Real-time systems**: RMS/EDF scheduling, WCET, **priority inversion/inheritance**, schedulability, deadlines & jitter
- **Real-time Linux (PREEMPT_RT)**: SCHED_FIFO/DEADLINE, CPU isolation/affinity, latency measurement (cyclictest)
- **Embedded programming**: ARM Cortex-M, bare-metal, interrupts/DMA, memory-mapped I/O, device drivers; an **RTOS** (FreeRTOS/Zephyr)
- **OS & computer-architecture** working knowledge: processes/threads, memory hierarchy, caches, SoC/heterogeneous compute
- **On-robot deployment**: cross-compilation, sensor/actuator bring-up, **ros2_control**, Docker/containers on Jetson-class targets
- **Profiling & determinism**: latency/jitter measurement, tracing (LTTng / `ros2_tracing`), zero-copy transport

**Preferred / differentiating**
- **Deterministic ROS 2 execution** — real-time executors (rclc budget-based, **PiCAS**), executor/DAG-chain response-time analysis
- **micro-ROS** on MCUs (Micro XRCE-DDS, rclc executor) in <100 KB RAM
- **Zenoh / `rmw_zenoh`** and DDS↔Zenoh bridging for multi-robot / fleet / WAN
- **GPU-accelerated ROS 2**: Isaac ROS, **NITROS** type-negotiation, **iceoryx** shared-memory zero-copy, CUDA pipelines
- **Edge-AI optimization**: quantization / pruning / distillation, **TensorRT / TVM / ONNX Runtime / TF-Lite Micro**
- **On-robot foundation-model (VLA/VLM/LLM) inference** — chunked/async execution, latency budgeting, Jetson Thor / JetPack
- **Real-time networking / fieldbuses**: EtherCAT, CAN/CAN-FD, **TSN**; safety-critical / mixed-criticality (ISO 26262, DO-178C, real-time hypervisors)
- Publications at **RTSS / RTAS / EMSOFT / ECRTS**; contributions to ROS 2 / rmw / micro-ROS

**Tools & tech stack** — *Languages:* C, C++, Python, **Rust** (emerging) · *Middleware:* ROS 2, DDS (Cyclone / Fast DDS / RTI Connext), **Zenoh / rmw_zenoh**, micro-ROS / Micro XRCE-DDS · *Real-time:* **PREEMPT_RT Linux**, **FreeRTOS, Zephyr**, NuttX, QNX, Xenomai · *Control/exec:* ros2_control, rclc/rclcpp executors · *Zero-copy/accel:* **iceoryx, NITROS**, CUDA · *Edge inference:* **TensorRT, TVM, ONNX Runtime, TF-Lite Micro**, Triton · *Buses:* EtherCAT, CAN/CAN-FD, TSN, real-time Ethernet · *Observability:* **MCAP, Foxglove, Rerun**, LTTng / ros2_tracing · *Hardware/targets:* **NVIDIA Jetson (Orin / Thor)**, STM32 / Teensy / ESP32, Raspberry Pi, F1TENTH · *Sim:* Gazebo, Isaac Sim / Isaac ROS.

## 7. Trending Research Topics `⏱ as of mid-2026`
> **Time-sensitive — re-research on a schedule.** The "SAM2/DINO" of this niche isn't one model — it's the substrate everyone builds on: **ROS 2 + a pluggable, Zenoh-forward RMW**, **micro-ROS** on MCUs, and **NVIDIA Jetson Thor + Isaac ROS / NITROS zero-copy** as the edge plumbing under the foundation-model era. `🆕` = 2026 release; treat vendor/benchmark claims as author-reported.

- **ROS 2 middleware evolution — Zenoh's rise over DDS:** **rmw_zenoh / Eclipse Zenoh** became a **Tier 1** ROS 2 middleware in **Kilted Kaiju (May 2025)** alongside Zenoh 1.0 — a data-centric protocol unifying pub/sub, query, and storage, built for lossy/WAN/constrained links; the presumptive future default · incumbents **Cyclone DDS / Fast DDS (eProsima) / RTI Connext** remain the production LAN standard (Cyclone the common default) · **zenoh-plugin-ros2dds** (Zenoh↔DDS bridge) = the pragmatic fleet/cloud migration path without touching the RMW · **ROS 2 Kilted Kaiju (May 2025)** → 🆕 **Lyrical Luth (May 2026)** carry the Zenoh + real-time work · benchmarks: Zenoh ~2× DDS throughput in multi-machine/zonal topologies, DDS keeps best single-hop latency/QoS.
- **Real-time Linux & deterministic ROS 2 execution:** **PREEMPT_RT mainlined in Linux 6.12 (Sept 2024)** — the ~20-year effort landed; real-time Linux is now default-available (x86/ARM64/RISC-V), the base for on-robot control · 🆕 **"A Survey of Real-Time Support, Analysis, and Advancements in ROS 2"** — the anchor reference consolidating scheduling analysis, executor timing, and middleware determinism · **ROSRT** (RTSS 2025 — per-callback priority/deadline on arbitrary ROS graphs) · 🆕 **ReDAG-RT** (global rate-priority scheduling for real-time multi-DAG execution) · **LaME** (RTNS 2025), **CIL-EDF**, **ROSGuard** (bandwidth regulation) — deadline-driven multi-threaded-executor enhancements fixing priority inversion · **ros2_control** the de-facto real-time controller substrate (SCHED_FIFO loop, hardware-interface plugins).
- **micro-ROS & embedded / MCU middleware:** **micro-ROS** — client-agent bridge over **Micro XRCE-DDS**, runs on FreeRTOS / Zephyr / bare-metal in ~60 KB RAM; the standard for time-critical sensors/actuators · 🆕 **Meta-ROS** (proposed next-gen middleware architecture for adaptive/scalable robotic systems) · **budget-based real-time Executor for micro-ROS** (deterministic execution on MCUs) · **Zenoh-Pico / Micro XRCE-DDS** pushing the new middleware down to constrained devices.
- **GPU-accelerated & zero-copy transport (edge-compute plumbing):** **NITROS (NVIDIA Isaac Transport for ROS)** — type adaptation + negotiation for **zero-copy GPU-to-GPU** message flow across ROS 2 nodes, the core of accelerated perception graphs · **Eclipse iceoryx** — shared-memory true zero-copy IPC underpinning large-pointcloud pipelines (Autoware) and DDS zero-copy · **PAAM** (coordinated, priority-driven GPU-accelerator management in ROS 2) · **HPRM** (high-performance robotic middleware, low-latency IPC).
- **The edge hardware + stack layer — NVIDIA's physical-AI platform:** **Jetson Thor / IGX Thor (2025)** — 2070 FP4 TFLOPS, 128 GB, 40–130 W, ~7.5× AGX-Orin AI perf, explicitly built to run VLA/LLM/VLM on-robot · **Isaac ROS + Holoscan** — GPU-accelerated ROS 2 perception (VSLAM, detection, depth) + sensor-processing SDK, the deploy stack on Thor/Orin · **GR00T N1.5 running on-device on Thor** — the concrete "foundation model on the robot" milestone tying middleware to the VLA era.
- **On-robot foundation-model inference — the live frontier:** making 1–7B VLAs run in real time on-robot is the hottest 2025–26 systems problem — **Real-Time Chunking (RTC)** (Physical Intelligence; inference-time async execution of flow/diffusion VLAs via freeze-and-inpaint action chunks, robust to inference delay, no retraining) · 🆕 **FASTER: Rethinking Real-Time Flow VLAs** (Horizon-Aware Schedule compresses reaction latency to a single flow step) · 🆕 **Masked Action Chunking** / **Leave No Observation Behind** (2025) — real-time correction/execution of chunks under latency · **SmolVLA** (2025 — compact VLM + flow-matching action expert; trains on 1 GPU, runs on CPU/consumer GPU) · **Lite VLA** / 🆕 **LiteVLA-Edge** (4-bit GGUF on-device control, ~150 ms latency on Orin-class) · 🆕 **VLA-Perf** ("How Fast Can I Run My VLA?") & **Characterizing VLAs across XPUs** — the emerging on-robot-inference benchmarking / analytical-modeling layer · **TensorRT Edge-LLM** (FP8 / NVFP4 / INT4 + KV-cache) the quantization toolchain everyone deploys through.
- **Data infrastructure & observability:** **MCAP** — self-describing log container, **rosbag2's default format since ROS 2 Iron**, the portable standard for multimodal robot data · **Foxglove / Rerun** — the visualization + observability tools of record for ROS 2 / MCAP pipelines.

*Net trend: the middleware/RT/embedded stack is consolidating around **ROS 2 with a pluggable, Zenoh-forward RMW and mainline PREEMPT_RT determinism**, while **NVIDIA Jetson Thor + Isaac ROS / NITROS zero-copy** becomes the edge substrate — all now bending toward one goal: running big VLA foundation models on-robot in real time via chunked/async inference and aggressive quantization. Several 2026 items are fresh preprints; re-verify at publish.*
