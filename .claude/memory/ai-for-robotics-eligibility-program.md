Building the eligibility/criteria + training program (courses, syllabi, labs, hardware, partnerships) for engineers to work on "AI for robotics," mapped per niche across undergrad → postgrad → experience/projects. Final deliverable = a shareable **website** (published as both a Claude Artifact and GitHub Pages). The user drives this project and wants maximal comprehensiveness ("don't miss anything").

**Structure decision (backbone, confirmed by verified deep-research):** two-axis taxonomy — **Axis A** technical capability niches (the spine: perception, SLAM, planning, control, manipulation, robot learning, VLAs/foundation models, simulation, HRI, systems/safety) × **Axis B** industry verticals as *overlays* that add domain + regulation + hardware + safety envelope (NOT parallel per-industry curricula — avoids N×M explosion). "Physical AI" = framing/branding only, not a niche. OPEN: whether embodiment/platform (humanoid/AMR/drone/AUV) becomes a third axis vs. a tag.

**Renderer lesson (load-bearing):** `site/src/build-site.js` rendered emphasis with regexes, which silently corrupted **16 of the 49 already-published profiles** — a lazy `/\*\*\*(.+?)\*\*\*/` matched from one `***` run to the next on the same line, bold-wrapping-italic produced mis-nested `</strong></em>`, and escaped `RRT\*` leaked into the rules. Replaced with `emphasize()`, a real **CommonMark delimiter-stack parser (incl. the rule-of-three)**, plus fenced-code-block support. **Do not regress this to regexes.** Verify site changes by diffing the rendered JSON payload against a saved baseline build, and by counting stray `**` / raw backticks / unbalanced tags across all profiles.
